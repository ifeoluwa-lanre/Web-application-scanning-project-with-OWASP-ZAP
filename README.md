# Web-application-scanning-project-with-OWASP-ZAP ~ In Progress

This project demonstrates a comprehensive (DAST) workflow using OWASP ZAP to identify and validate web-based vulnerabilities. The assessment integrates automated scanning with manual security testing to simulate a real-world Application Security audit, ensuring the integrity of web applications within a secure Software Development Life Cycle.

# Core Objectives
- Vulnerability Discovery: Identifying critical flaws, including SQL Injection (SQLi), Cross-Site Scripting (XSS), and Broken Access Control.
- Methodology Integration: Combining DAST with SAST/SCA tools to provide a multi-layered security analysis of application source code and dependencies.
- Technical Validation: Performing manual verification of automated scan results to eliminate false positives and assess business risk.
- Incident Contextualization: Connecting AppSec findings to network-level indicators to support SOC/Incident Response investigations.

# Technology used
For this project, the technology used:
- OWASP ZAP; Scanner
- Acunetix; Vulnerable Site

# Outcomes
- Reconnaissance & Spidering: Mapping the application attack surface.
- Active Scanning: Executing targeted attacks against input vectors to discover high-risk vulnerabilities.
- Triage & Interpretation: Analyzing raw scan data to prioritize issues based on severity.
- Documentation: Authoring executive-level reports that bridge the gap between technical findings and business impact.
